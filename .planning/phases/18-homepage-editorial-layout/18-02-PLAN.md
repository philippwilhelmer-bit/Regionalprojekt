**HomepageLayout.tsx** ("use client"):
- Props: `{ hero: ArticleWithBezirke | null; pinnedArticles: ArticleWithBezirke[]; allArticles: ArticleWithBezirke[] }`
- On mount (useEffect), read `localStorage.getItem("bezirk_selection")`, parse as string[], store in state `selectedSlugs`.
- State: `selectedSlugs: string[]` (empty initially = show all), `mounted: boolean` (false until useEffect runs).
- Derive filtered data from props + selectedSlugs:
  - `filteredPinned`: if selectedSlugs is empty, use all pinnedArticles. If set, filter to articles that have a bezirk with slug in selectedSlugs OR have isStateWide=true.
  - `bezirkSections`: if selectedSlugs is empty (or not mounted yet), do NOT group — show one flat editorial grid of all articles. If selectedSlugs is set, call `groupArticlesByBezirk(allArticles)` and filter to only the selected bezirk slugs.
- Rendering order:
  1. `{hero && <HeroArticle article={hero} />}` — always show if exists
  2. `{filteredPinned.length > 0 && <TopMeldungenRow articles={filteredPinned} />}`
  3. `<AdUnit zone="hero" />` — preserve existing ad slot, placed between top-stories and sections
  4. Bezirk sections OR flat grid:
     - If selectedSlugs set and mounted: iterate `bezirkSections` map, render `<BezirkSection>` for each with `showDivider={index > 0}`. Insert `<AdUnit zone="between-articles" />` after every 2nd section.
     - If no selection: render a single `<BezirkSection bezirkName="Alle Nachrichten" articles={allArticles.slice(0, 9)} showDivider={false} />` followed by remaining articles in simple ArticleCard list (to keep page manageable).
- Show heading above sections: if bezirk selected show "Mein Bezirk" in `font-headline text-lg font-semibold`, else "Alle Nachrichten".
- Pre-mount state: show hero + pinned (unfiltered) + all articles unfiltered. After mount, if bezirk selected, re-derive filtered views. This avoids hydration mismatch.
- Empty state: if allArticles is empty, show "Noch keine Nachrichten" message with option to clear bezirk selection (same pattern as existing ArticleFeed).
