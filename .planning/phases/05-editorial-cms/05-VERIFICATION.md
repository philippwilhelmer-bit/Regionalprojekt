**Note on auth gap (Warning):** The middleware guards page-level navigation to `/admin/**`. However, Next.js 15 Server Actions are invoked via POST requests to a specific action endpoint. When called directly (not via page navigation), they may not trigger the middleware. The layout.tsx provides a secondary server-side session check for page rendering, but mutation Server Actions (togglePin, softDelete, etc.) in articles/exceptions/sources do not independently verify the session. This is flagged as a warning because: (1) the middleware does cover the vast majority of attack surface, (2) the admin URL structure is not public-facing, and (3) the plan notes document this as a Phase 6 TODO. It does not block the phase goal but is a known security weakness.
