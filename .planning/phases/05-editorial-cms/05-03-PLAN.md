  <behavior>
    CMS-01 createManualArticle:
    - Input: { title, content, bezirkIds: number[], seoTitle?, metaDescription?, isPinned?, isFeatured? }
    - Creates Article with source=MANUAL, isAutoGenerated=false, status=PUBLISHED, publishedAt=now
    - Creates ArticleBezirk rows for each bezirkId
    - Returns created article

    CMS-02 updateArticle:
    - Input: { id, title?, content?, seoTitle?, metaDescription?, bezirkIds? }
    - Updates specified fields on the Article row
    - If bezirkIds provided: deletes existing ArticleBezirk rows then creates new ones (atomic replacement)

    CMS-02 togglePin:
    - Input: articleId
    - Reads current isPinned, sets to opposite value

    CMS-02 toggleFeature:
    - Input: articleId
    - Reads current isFeatured, sets to opposite value

    CMS-02 softDelete:
    - Input: articleId
    - Sets Article.status = REJECTED

    CMS-01/02 listArticlesAdmin:
    - Supports optional filters: bezirkId, sourceType (ArticleSource), status (ArticleStatus or array), fromDate, toDate
    - Default: excludes REJECTED and FAILED (unless explicitly requested)
    - Returns articles with bezirke relation included
    - Ordered by publishedAt desc, createdAt desc
    - Supports limit (default 50) and offset (default 0)

    CMS-03 approveArticle:
    - Input: articleId
    - Sets Article.status = PUBLISHED

    CMS-03 rejectArticle:
    - Input: articleId
    - Sets Article.status = REJECTED

    CMS-03 listExceptionQueue:
    - Returns articles with status=REVIEW only
    - Includes rawPayload field (for side-by-side display)
    - Ordered by createdAt asc (oldest first)
  </behavior>
