Add `seedBulkArticles(prisma: PrismaClient, count: number): Promise<void>` to `src/test/setup-db.ts` after the existing `cleanDb` function. Import `randomUUID` from `node:crypto` at the top of the file (add to existing imports).

Implementation:
1. Seed all 13 Steiermark Bezirke using `prisma.bezirk.upsert` with `where: { slug }` — import the Bezirk data directly inline (slug/name/gemeindeSynonyms). Use the same 13 Bezirke from `prisma/seed-data/bezirke.ts` (read that file to get the exact data).
2. Fetch all Bezirk records: `const bezirke = await prisma.bezirk.findMany()`
3. For each `n` from 0 to count-1:
   - `prisma.article.create` with:
     - `publicId: randomUUID()` (non-null — required for detail lookup test)
     - `source: 'MANUAL'`
     - `title: \`Artikel ${n}\``
     - `content: \`Inhalt für Artikel ${n}\``
     - `status: 'PUBLISHED'`
     - `publishedAt: new Date()`
     - `isAutoGenerated: false`
   - Then `prisma.articleBezirk.create` with `{ articleId: article.id, bezirkId: bezirke[n % bezirke.length].id }` — assigns one Bezirk per article in round-robin for even distribution

Performance note: Use a simple sequential loop — pgLite is WASM-based and sequential inserts are fine. Do NOT use `createMany` as pgLite support may vary. The caller (Plan 02's Criterion 4 `beforeAll`) seeds once, not per-test.

NOTE: Article has NO `slug` field — omit it entirely from the create call. The `publicId` default in schema is `nanoid()` but we override with `randomUUID()` explicitly to get a non-null UUID.
