## Objective

<objective>
Build the three Server-Action-Trinity triplets that orchestrate Phase 47's import + batch-geocode pipeline. All sit in a single new file `src/lib/admin/doctors-import-actions.ts` (sibling to Phase 46's `doctors-actions.ts` per RESEARCH project structure):

1. **parseAndPreviewCsv{Db,Action,Form}** — accepts uploaded CSV bytes, validates with `parseDoctorsCsv`, resolves Bezirk names → bezirkIds via DB lookup, classifies rows as new/update/conflict against existing arztNr keys, caches the typed `ParsedRow[]` keyed by a UUID token, returns `{token, summary, conflicts}`.
2. **commitCsvImport{Db,Action,Form}** — accepts a token, loads parsed rows from in-memory cache, performs all upserts inside a single `db.$transaction([...])` with strict editorial-field allow-list on update payloads, sets lat/lon/mapImageUrl=null when address changed.
3. **geocodeBatch{Db,Action,Form}** — selects up to 200 doctors WHERE lat IS NULL, sequentially geocodes each with `await sleep(1100)`, generates map image, updates the row inline. Returns `{processed, remaining}`.

Purpose: This is the data-flow heart of Phase 47. Every D-13..D-22 decision is implemented here. Plans 47-04 (admin UI) and 47-05 (public UI) consume these actions but do not modify the pipeline.

Output: One actions file, one test file (covers DIR-23, DIR-24, DIR-25, DIR-26 + sleep timing via `vi.useFakeTimers`), in-memory PREVIEW_CACHE with 15-min TTL pruning.
</objective>
