<task type="auto">
  <name>Task 1: Create railway.toml and update package.json start script</name>
  <files>railway.toml, package.json</files>
  <action>
1. Create `railway.toml` at repo root with the following configuration:

```toml
[build]
buildCommand = "npx prisma generate && npm run build"

[deploy]
startCommand = "npx prisma migrate deploy && next start -p ${PORT:-3000}"
healthcheckPath = "/"
healthcheckTimeout = 120
restartPolicyType = "ON_FAILURE"
restartPolicyMaxRetries = 3
```

CRITICAL: `prisma migrate deploy` goes in startCommand, NOT buildCommand. Railway's internal network (where Postgres lives) is only available at runtime, not build time. `prisma generate` is safe in buildCommand because it needs no DB connection.

2. Update `package.json` — change the `start` script from `next start` to `next start -p ${PORT:-3000}`. This ensures Railway's dynamic PORT is respected. The `-p` flag binds Next.js to Railway's assigned port. The `:-3000` fallback preserves local dev behavior.

Do NOT add `output: "standalone"` to next.config.ts — per research, standard `next start` is simpler and avoids touching multiple files. Standalone is an optimization, not a requirement.
  </action>
  <verify>
    <automated>grep -q "prisma generate" railway.toml && grep -q "prisma migrate deploy" railway.toml && grep -q "healthcheckPath" railway.toml && node -e "const p=require('./package.json'); if(!p.scripts.start.includes('PORT')) process.exit(1)" && echo "PASS"</automated>
  </verify>
  <done>railway.toml exists with correct build/start separation (generate in build, migrate in start), health check config, and restart policy. package.json start script includes PORT binding.</done>
</task>

2. Then import and run the AI pipeline (from `src/scripts/ai-run.ts` main logic)
3. Log start/end timestamps and overall success/failure
4. Exit with code 0 on success, code 1 on fatal error
5. Handle errors gracefully — if ingest fails, still attempt AI pipeline (AI processes previously fetched articles)

Structure:
```typescript
/**
 * Combined Railway cron entry point.
 * Runs ingest pipeline, then AI pipeline, sequentially.
 * Designed for Railway cron service: runs on schedule, exits cleanly.
 *
 * Usage: npx tsx scripts/railway-cron.ts
 * Railway cron schedule: 0 */2 * * * (every 2 hours)
 */
```

IMPORTANT: Do NOT import from `scripts/ingest-run.ts` or `src/scripts/ai-run.ts` directly (they are CLI entry points with top-level execution). Instead, import the underlying functions:

Mirror the logic from both scripts but in a single sequential flow with combined logging.
