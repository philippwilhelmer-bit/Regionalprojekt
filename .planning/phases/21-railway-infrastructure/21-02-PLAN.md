<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Create Railway project, provision PostgreSQL, set environment variables, and deploy</name>
  <action>
The user must perform the following steps in the Railway dashboard. Claude cannot do these because Railway project creation and service configuration require authenticated dashboard access.

**Step 1: Create Railway Project**
1. Go to https://railway.com/dashboard → "New Project"
2. Name it `regionalprojekt`

**Step 2: Add PostgreSQL Addon**
1. In the project, click "+ New" → "Database" → "PostgreSQL"
2. Ensure region is set to EU West (europe-west4)

**Step 3: Connect GitHub Repository**
1. Click "+ New" → "GitHub Repo"
2. Select the `Regionalprojekt` repository
3. Branch: `main` (auto-deploy on push)
4. DO NOT deploy yet — set variables first

**Step 4: Set Environment Variables on the Web Service**
Go to web service → "Variables" tab and set ALL of these BEFORE the first deploy:

```
DATABASE_URL = ${{Postgres.DATABASE_URL}}
ANTHROPIC_API_KEY = <your Anthropic API key>
ADMIN_PASSWORD = <choose a password for CMS login>
ADMIN_SESSION_SECRET = <run: node -e "console.log(require('crypto').randomBytes(32).toString('hex'))">
NEXT_PUBLIC_BASE_URL = <will be set after domain is generated in Step 5>
NEXT_PUBLIC_IS_TEST_SITE = true
```

Note: Do NOT set OTS_API_KEY or NEXT_PUBLIC_ADSENSE_PUB_ID (intentionally omitted for test site).

**Step 5: Generate Public Domain**
1. Go to web service → "Settings" → "Networking"
2. Click "Generate Domain" to get a `*.up.railway.app` URL
3. Copy the full URL (e.g., `https://regionalprojekt-production.up.railway.app`)
4. Go back to Variables and set `NEXT_PUBLIC_BASE_URL` to this URL

**Step 6: Trigger Deploy**
1. Railway should auto-deploy after variables are set and GitHub is connected
2. If not, click "Deploy" manually or push a commit to main

**Step 7: Wait for Build + Deploy**
1. Watch the deploy logs in Railway dashboard
2. Build should show: `prisma generate` then `next build`
3. Start should show: `prisma migrate deploy` then `next start`
4. Health check should pass (homepage returns 200)

**Step 8: Seed the Database**
After deploy succeeds, run the seed command via Railway CLI:
```bash
# Install Railway CLI if not already installed
npm install -g @railway/cli

# Login to Railway
railway login

# Link to the project
railway link

# Run seed against Railway's database
railway run npx prisma db seed
```

**Step 9 (Optional): Set Up Cron Service**
1. In Railway project, click "+ New" → "GitHub Repo" (same repo)
2. Name this service `cron`
3. Set Variables:
   ```
   DATABASE_URL = ${{Postgres.DATABASE_URL}}
   ANTHROPIC_API_KEY = <same key as web service>
   ```
4. In Settings:
   - Start command: `npx tsx scripts/railway-cron.ts`
   - Cron schedule: `0 */2 * * *` (every 2 hours)
   - Region: EU West
