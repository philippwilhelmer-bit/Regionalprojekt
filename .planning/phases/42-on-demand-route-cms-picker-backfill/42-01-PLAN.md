    - backfillMapImages only processes PUBLISHED articles with imageUrl null

  <action>
    **1. Create `src/lib/admin/map-actions.ts`** — Server Actions file with 'use server' directive:

    - `generateMapForArticle(articleId: number): Promise<{ url: string; credit: string } | { error: string }>` — calls requireAuth(), loads article from prisma, runs extractLocation on `[article.title, article.content].filter(Boolean).join('\n\n')`, falls back to llmLocationFallback with new Anthropic() client, geocodes via geocodeLocation(prisma, locationName), generates via generateMapImage(geo.lat, geo.lon, article.title ?? '', articleId, geo.locationType), persists with prisma.article.update({ where: { id: articleId }, data: { imageUrl, imageCredit } }), returns { url, credit }. Each step that fails returns { error: '...' } with a descriptive message.

    - `backfillMapImages(): Promise<BackfillResult>` — calls requireAuth(), queries prisma.article.findMany with `where: { status: 'PUBLISHED', imageUrl: null }, select: { id: true, title: true, content: true }, orderBy: { publishedAt: 'desc' }, take: 10`. Iterates sequentially (NOT Promise.all — Nominatim 1 req/s policy). For each article: extract location, geocode, generate map, update DB. Counts succeeded/failed/skipped. Returns { processed, succeeded, failed, skipped }.

    - Export the `BackfillResult` interface: `{ processed: number; succeeded: number; failed: number; skipped: number }`

    **2. Create `src/app/api/admin/generate-map/route.ts`** — POST route handler:

    - `export const maxDuration = 60` (tile fetch + sharp can take 8-15s)
    - Auth: `request.headers.get('authorization') !== Bearer ${process.env.CRON_SECRET}` returns 401
    - Parse body as `{ articleId?: number }`, validate articleId is present and numeric
    - Load article via prisma.article.findUnique, return 404 if not found
    - Run same pipeline as generateMapForArticle: extractLocation -> llmLocationFallback -> geocodeLocation -> generateMapImage
    - Persist result with prisma.article.update
    - Return `{ url, credit }` on success, `{ error }` with appropriate status codes (400, 404, 422, 500) on failure

    **3. Write tests using vi.mock for all external dependencies:**

    For route.test.ts: Mock prisma, extractLocation, llmLocationFallback, geocodeLocation, generateMapImage. Test the route handler by importing POST and calling it with constructed NextRequest objects. Mock `process.env.CRON_SECRET = 'test-secret'`.

    For map-actions.test.ts: Mock prisma, extractLocation, llmLocationFallback, geocodeLocation, generateMapImage, requireAuth. Test generateMapForArticle and backfillMapImages with various mock return values.

    IMPORTANT: Use `vi.mock()` at module level (not inline). For prisma, mock `@/lib/prisma` to return a mock object with `article.findUnique`, `article.findMany`, `article.update` as vi.fn(). For requireAuth, mock `./auth-node` to return a no-op. Do NOT use pglite for these tests — vi.mock with Prisma stubs is simpler and faster.

    Anti-patterns to avoid:
    - Do NOT call requireAuth() in the route handler (uses next/headers which is unavailable in Route Handlers in Next.js 15). Use CRON_SECRET bearer token auth pattern instead.
    - Do NOT use Promise.all in backfill — sequential only for Nominatim rate limits.
    - Do NOT forget `imageUrl: null` in the backfill findMany filter — this prevents overwriting Unsplash images.
    - Use `article.title + article.content` for location extraction text (NOT article.rawPayload).
