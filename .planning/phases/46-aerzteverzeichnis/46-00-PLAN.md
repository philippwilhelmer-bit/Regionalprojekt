  </behavior>
  <action>
    Refactor `generateMapImage` to accept an optional sixth parameter `options?: { pathPrefix?: string }`:

    1. In `src/lib/images/mapgen.ts`:
       - Change `uploadToBlob(articleId: number, imageBuffer: Buffer)` to `uploadToBlob(id: number, imageBuffer: Buffer, pathPrefix?: string)`. Body MUST guard against empty string explicitly:
         ```typescript
         async function uploadToBlob(id: number, imageBuffer: Buffer, pathPrefix?: string): Promise<string> {
           const prefix = pathPrefix || 'article'  // `||` not `??` — empty string MUST also fall back
           const blob = await put(`maps/${prefix}-${id}.jpg`, imageBuffer, { ... })
         }
         ```
         DO NOT rely on a JS default parameter (`pathPrefix: string = 'article'`) — defaults fire only for `undefined`, not `''`. An empty string would produce the broken path `maps/-7.jpg`. The `|| 'article'` expression covers both cases.
       - Change `generateMapImage` signature to:
         ```typescript
         export async function generateMapImage(
           lat: number,
           lon: number,
           headline: string,
           id: number,
           locationType?: string,
           options?: { pathPrefix?: string },
         ): Promise<MapImage | null>
         ```
         (rename `articleId` → `id` inside the function body; update warn-log too).
       - Pass `options?.pathPrefix` to `uploadToBlob` (let the helper apply the `|| 'article'` fallback once).
       - Update JSDoc to document the new parameter; note that `pathPrefix` defaults to `'article'` (covers both undefined and empty-string).

    2. In `src/lib/images/mapgen.test.ts`:
       - Add three test cases under the existing test file's `describe('generateMapImage', ...)` block (or a new describe if cleaner):
         (a) default call → assert Blob path starts with `maps/article-`
         (b) call with `{ pathPrefix: 'doctor' }` → assert Blob path starts with `maps/doctor-`
         (c) call with `{ pathPrefix: '' }` → assert Blob path falls back to `maps/article-` (verifies the `|| 'article'` guard, NOT a JS-default-param semantics test)

    AGENTS.md anti-bloat: do NOT introduce a new wrapper function like `generateDoctorMap`. Extend the existing API once; both callers pass through.
