## Integration Findings

### BROKEN: E2E Flow — "Deploy new Bundesland → all branding updates"

**Affected requirements:** CONF-01, CONF-02, READ-06

The Bundesland config file (`bundesland.config.ts`) declares `siteName`, `tagline`, `primaryColor`, `secondaryColor`, and `logoPath` but **none of these fields are read by any component**. Four source files hardcode `"Ennstal Aktuell"`:

| File | Line | Hardcoded value | Should use |
|------|------|-----------------|------------|
| `src/components/reader/Header.tsx` | 47 | `"Ennstal Aktuell"` | `config.siteName` |
| `src/app/layout.tsx` | 6 | `title: "Ennstal Aktuell"` | `config.siteName` |
| `src/lib/reader/rss.ts` | 19 | `"Ennstal Aktuell"` | `config.siteName` |
| `src/app/admin/login/page.tsx` | 7 | `"Ennstal Aktuell Admin"` | `config.siteName` |

A new Bundesland deployment that only changes `bundesland.config.ts` would still show "Ennstal Aktuell" in the browser tab, site header, RSS feed title, and admin login page.

**Config fields declared but consumed nowhere:** `primaryColor`, `secondaryColor`, `logoPath`.
