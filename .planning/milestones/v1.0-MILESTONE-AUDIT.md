## Integration Check (33/35 wired)

Cross-phase integration verified by integration checker agent. 33 of 35 requirement integration paths are fully wired.

### Integration Issues

**1. READ-02 — /bezirk/[slug] ArticleFeed override (medium severity)**
`BezirkPage` renders `ArticleFeed` with correct SSR articles for that Bezirk. On client mount, `ArticleFeed` reads `localStorage["bezirk_selection"]` and replaces the articles with the user's global preference. No `forcedBezirkSlugs` prop exists to override this behavior.
- Files: `src/app/(public)/bezirk/[slug]/page.tsx`, `src/components/reader/ArticleFeed.tsx`
- Fix: Pass a `forcedBezirkSlugs` prop from BezirkPage to bypass localStorage on mount

**2. READ-05 — Impressum TODO placeholders (high severity for go-live)**
`bundesland.config.ts` contains 4 `TODO:` placeholder strings that render verbatim on the Impressum page: telefon, unternehmensgegenstand, blattlinie, datenschutzEmail. The code wiring is correct — this is an operator data entry task.
- File: `bundesland.config.ts` lines 15-18
- Fix: Replace TODO values with real publisher data before production launch
