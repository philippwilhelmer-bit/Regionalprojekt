# Phase 6: Reader Frontend - Context

**Gathered:** 2026-03-22
**Status:** Ready for planning

<domain>
## Phase Boundary

The public-facing reader site: Bezirk selection and personalized feed, article detail pages, per-Bezirk RSS feeds, Google AdSense integration with GDPR consent, SEO-complete article pages (Open Graph, JSON-LD, sitemap.xml), and a legally compliant Austrian Impressum + Datenschutzerklärung. Admin/CMS routes are out of scope (Phase 5).

</domain>

<decisions>
## Implementation Decisions

### Bezirk selection flow
- **First-visit modal** — appears automatically on first visit with no prior localStorage selection
- **Multiple Bezirke allowed** — chip grid showing all 13 Bezirke; reader can select any combination
- **"Alle Bezirke" toggle** — one-tap option to subscribe to all 13 regions; stored as all-selected in localStorage
- **Skip button ("Später")** — modal is dismissable without selecting; no forced onboarding
- **Default state (no selection)** — homepage shows all Steiermark feed ("Alle Nachrichten") as fallback
- **Re-opening the selector** — tapping the Bezirk indicator in the top header re-opens the selection modal at any time
- **Persistence** — selection stored in localStorage; no account needed (READ-01)

### Navigation structure
- **Top header** — site name (from bundesland config) + Bezirk indicator (e.g. "Ennstal Aktuell · Liezen ▾"); tap indicator opens Bezirk modal
- **Bottom nav** — single "Nachrichten" tab (feed); minimal nav, most navigation via header + links
- **Bezirk browse pages** — /bezirk/[slug] exists but is reached by tapping Bezirk badges on article cards, not from the nav
- **Footer** — appears on homepage; contains Impressum link and RSS feed links (per-Bezirk + all-Steiermark)
- **Bottom nav visible on all pages** including article detail

### Homepage & feed layout
- **Article cards** — each card shows: title, ~2-sentence excerpt, Bezirk badge(s), relative timestamp, Bezirk-colored gradient placeholder image header
- **Placeholder image** — Bezirk-colored gradient (each Bezirk gets a distinct color). Source image extraction + AI image generation deferred to a future phase.
- **Pinned articles (`isPinned=true`)** — sorted to the very top of the feed regardless of date
- **Featured articles (`isFeatured=true`)** — visually larger card treatment (hero-size) above regular cards
- **Feed heading** — "Mein Bezirk" when Bezirke are selected, "Alle Nachrichten" (or "Steiermark Aktuell") when not
- **Loading** — infinite scroll (listArticles() DAL already supports limit/offset)
- **Empty state** — "Noch keine Nachrichten für deine Bezirke" + fallback to a selection of general Steiermark articles
- **AI disclosure on cards** — subtle "Automatisch erstellt" badge on cards where `isAutoGenerated=true`
- **No separate breaking news strip** — pinned articles surface to top of feed; no horizontal scroll strip
- **AdSense in feed** — hero ad zone above the feed; one inline ad unit every 5 article cards

### Article detail page
- **URL** — `/artikel/[publicId]/[seo-slug]` (e.g. `/artikel/k7x9mn/liezen-unwetter-warnung`)
  - `publicId` — nanoid field added to Article schema (`publicId String @unique @default(nanoid())`); non-sequential, doesn't reveal article volume
  - `slug` — generated from `seoTitle` if set, otherwise from `title`; slugified German text
  - **Wrong slug + correct publicId** → 301 redirect to canonical URL
- **Content shown** — title, full article body, source attribution ("Quelle: OTS.at" or RSS feed name), AI disclosure label if `isAutoGenerated=true`, publication timestamp (exact date/time), related articles from same Bezirk (3–5 cards at bottom)
- **Breadcrumb** — Home > [Bezirk name] > [Article title]
- **Social share** — Web Share API button; falls back to copy-link button on desktop
- **AdSense** — one responsive ad unit after the article body
- **Bottom nav** — visible on article detail page (same as everywhere)

### Per-Bezirk RSS feeds (READ-06)
- **URL pattern** — `/rss/[bezirk-slug].xml` per Bezirk (e.g. `/rss/liezen.xml`) + `/rss/steiermark.xml` for all regions
- **Content** — full article body in each `<item>` entry (not excerpt)
- **Volume** — 20 most recent published articles per feed
- **AI disclosure in RSS** — not included; disclosure lives on the web UI only
- **Link in entry** — canonical article URL (`/artikel/[publicId]/[seo-slug]`)

### AdSense integration (AD-01)
- **Script loading** — `next/script` with `afterInteractive` strategy in root layout
- **Ad unit abstraction** — `<AdUnit zone="hero" | "between-articles" | "article-detail">` component; AdSense-specific code contained within it; swappable for other networks
- **Ad unit IDs** — configured via `.env`: `NEXT_PUBLIC_ADSENSE_PUB_ID`, `NEXT_PUBLIC_ADSENSE_HERO_SLOT`, `NEXT_PUBLIC_ADSENSE_INLINE_SLOT`, `NEXT_PUBLIC_ADSENSE_ARTICLE_SLOT`
- **Ad format** — responsive auto-sizing (`data-ad-format="auto"`) for all zones
- **Placement zones active:**
  - Hero: above the article feed on homepage
  - Between articles: one unit every 5 article cards in the feed
  - Article detail: one unit after the article body
- **No ads on** — Impressum page, /bezirk/[slug] browse pages
- **Cookie consent** — required (Austria = EU/DSGVO). Bottom banner with "Akzeptieren" / "Nur notwendige Cookies". Consent stored in localStorage; does not re-prompt after answered.
  - Banner links to Datenschutz section on `/impressum`
  - If rejected → AdSense served in non-personalized ads (NPA) mode (not hidden)

### SEO (SEO-01, SEO-03, SEO-04)
- **Open Graph meta tags** — on all article pages: `og:title` (seoTitle or title), `og:description` (metaDescription), `og:url` (canonical), `og:type: article`
- **JSON-LD NewsArticle schema** — on each article detail page; includes headline, datePublished, author (placeholder for automated), publisher (from bundesland config)
- **Sitemap.xml** — `/sitemap.xml` generated via Next.js route handler; queries all published article publicIds+slugs + static pages (Bezirk pages, Impressum); updated on each request (not pre-built)
- **Canonical URLs** — all article pages have `<link rel="canonical">` pointing to the /artikel/[publicId]/[seo-slug] form

### Impressum + Datenschutzerklärung (READ-05)
- **URL** — `/impressum`
- **Structure** — combined page: Impressum (MedienG §25 + ECG §5) + Datenschutzerklärung (DSGVO)
- **Sections:**
  - Medieninhaber / Herausgeber — [MEDIENINHABER_NAME], [ADRESSE], [EMAIL], [TELEFON] placeholders
  - Unternehmensgegenstand — [UNTERNEHMENSGEGENSTAND] placeholder
  - Blattlinie — [BLATTLINIE] placeholder (editorial direction per MedienG §25 Abs. 4)
  - Haftungsausschluss — standard disclaimer
  - **KI-generierte Inhalte** — section explaining that some articles are automatically generated by AI, noting the "Automatisch erstellt" label, and that the operator reviews flagged content
  - Datenschutzerklärung — covers: cookies (localStorage for preferences, consent cookie), Google AdSense (third-party cookies), no user registration/accounts, contact for data requests: [DATENSCHUTZ_EMAIL] placeholder
- **Content** — Claude drafts complete template with all MedienG/ECG required fields as labeled placeholders; operator fills in real data before launch

### Claude's Discretion
- Exact Bezirk color palette (one distinct color per Bezirk)
- Tailwind CSS component design (typography, spacing, shadows) following alpine-themed design reference
- Loading skeleton design for feed and article detail
- Exact relative timestamp formatting ("vor 2 Stunden" / "gestern" / exact date after 7 days)
- RSS XML generation library choice (rss, feed, or custom)
- Slug generation / slugification library (slugify etc.)
- nanoid vs cuid2 for publicId generation
- Exact JSON-LD schema fields beyond the required set
- Related articles selection algorithm (same Bezirk, recency-ordered, exclude current)

</decisions>

<specifics>
## Specific Ideas

- **Bezirk modal** — chip grid showing all 13 Bezirke at once; "Alle Bezirke" toggle at top; "Später" dismiss button at bottom. Tapping header Bezirk indicator re-opens it.
- **Article cards** — Bezirk-colored gradient as card image header (distinct color per Bezirk); makes the feed visually differentiated by region without needing real photos
- **URL privacy** — non-sequential nanoid for publicId so article count is not exposed via URLs (e.g. `k7x9mn` not `42`)
- **AdSense NPA mode** — if reader rejects cookies, serve non-personalized ads rather than hiding slots entirely
- **Impressum** — combined with Datenschutzerklärung on one page; AI disclosure section specifically explains the automated generation pipeline and the "Automatisch erstellt" label

</specifics>

<code_context>
## Existing Code Insights

### Reusable Assets
- `src/lib/content/articles.ts` — `listArticles(bezirkId?, status?, limit?, offset?)`, `getArticleById()`, `getArticlesByBezirk(bezirkSlug)` — all ready for reader frontend; filter by status=PUBLISHED needed
- `src/lib/content/bezirke.ts` — `listBezirke()`, `getBezirkBySlug()`, `getBezirkById()` — all ready
- `src/app/(public)/` route group exists — correct location for all reader-facing routes
- Prisma schema already has: `isPinned`, `isFeatured`, `seoTitle`, `metaDescription`, `isAutoGenerated`, `publishedAt`, `source`, `ArticleStatus` enum
- `src/app/(admin)/` route group shows the pattern — reader routes follow the same `(public)` group pattern

### Established Patterns
- TypeScript overloads + duck-typing `$connect` for DI — all new server-side DAL functions follow this
- `listArticles()` already accepts `bezirkId` and `status` filter — add `status: 'PUBLISHED'` filter for public reader queries
- Next.js App Router pattern established in Phase 5 — Server Components for data fetching, Server Actions for mutations, `next/script` for third-party scripts

### Integration Points
- **New `publicId` field** needed on `Article` schema — add migration + backfill for existing articles
- `listArticles()` needs extension or new variant for reader use: filter `status=PUBLISHED`, support `isPinned`/`isFeatured` sort ordering
- `/rss/[slug].xml` routes use `getArticlesByBezirk()` DAL — returns full article with all fields
- `bundesland.config.ts` has ad zone definitions (`AdZone` — hero, between-articles, article-detail) — AdUnit component reads zone props and maps to `.env` slot IDs
- Article detail page reads from `getArticleById()` — extend to include Bezirk info for breadcrumb and related articles query
- `src/app/page.tsx` is currently the Next.js default scaffold — replace entirely with the reader homepage

</code_context>
