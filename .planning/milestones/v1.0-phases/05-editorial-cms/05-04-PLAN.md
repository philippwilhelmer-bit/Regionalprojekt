```typescript
export async function getAiConfig(db?: PrismaClient): Promise<AiConfig> {
  const client = (db && '$connect' in db) ? db : prisma
  const existing = await client.aiConfig.findFirst()
  if (existing) return existing
  return client.aiConfig.create({
    data: { tone: 'NEUTRAL', articleLength: 'MEDIUM' }
  })
}
```

  modelOverride: string | null
}

<!-- listSourcesAdmin health dashboard data -->
Per-source data needed: healthStatus, lastSuccessAt, latest IngestionRun (itemsFound, itemsNew),
count of Articles with status IN ('FAILED', 'ERROR') for this source.

Source model:
- source.type = ArticleSource enum (OTS_AT | RSS | MANUAL)
- source.runs = IngestionRun[] relation
- IngestionRun has: itemsFound Int, itemsNew Int, createdAt DateTime

Article → Source: Articles have source field (ArticleSource enum, not FK).
Note: Articles reference source TYPE not source ID (no FK). For FAILED+ERROR count per source,
filter by source type. If multiple sources have same type this is approximate — document this limitation.

DAL DI overload pattern: same as established ($connect duck-typing).

  <behavior>
    CMS-04 createSource:
    - Input: { url: string, type: ArticleSource, pollIntervalMinutes?: number }
    - Creates Source row with enabled=true, healthFailureThreshold=3 (default), healthStatus=OK
    - Returns created Source

    CMS-04 updateSource:
    - Input: { id: number, pollIntervalMinutes?: number, healthFailureThreshold?: number, enabled?: boolean }
    - Updates specified fields on Source row

    CMS-04 listSourcesAdmin:
    - Returns all sources with:
      - healthStatus, lastSuccessAt (from Source row)
      - Latest IngestionRun (itemsFound, itemsNew, createdAt)
      - FAILED+ERROR article count (articles with status IN [FAILED, ERROR] filtered by source type)

    AICONF-01 getAiConfig:
    - Find-or-create singleton: if no row exists, creates with tone=NEUTRAL, articleLength=MEDIUM
    - Returns AiConfig row

    AICONF-01 upsertAiConfig:
    - Input: Partial<{ tone, articleLength, styleNotes, modelOverride }>
    - If row exists: updates it. If not: creates it with provided values + defaults.
    - Returns updated AiConfig

    AICONF-02 getResolvedAiConfig:
    - Input: sourceId
    - Fetches global AiConfig (via getAiConfig) and AiSourceConfig for sourceId
    - Returns ResolvedAiConfig: per-source value if set, global value otherwise
    - NULL override fields fall through to global

    AICONF-02 upsertAiSourceConfig:
    - Input: { sourceId, tone?, articleLength?, styleNotes?, modelOverride? }
    - Creates or updates AiSourceConfig row for sourceId (upsert on unique sourceId)

    AICONF-02 deleteAiSourceConfig:
    - Input: sourceId
    - Deletes AiSourceConfig row for sourceId (resets per-source overrides to global)

    getPipelineConfig:
    - Find-or-create singleton: if no row exists, creates with maxRetryCount=3, deadManThresholdHours=6
    - Returns PipelineConfig row

    upsertPipelineConfig:
    - Input: Partial<{ maxRetryCount, deadManThresholdHours }>
    - Creates or updates the singleton PipelineConfig row
  </behavior>
