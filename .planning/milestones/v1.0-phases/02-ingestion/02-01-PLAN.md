  <files>prisma/schema.prisma, prisma/migrations/20260321_ingestion/migration.sql</files>
  <action>
Add the following to prisma/schema.prisma (append after the Article model):

1. New enum SourceHealth with values: OK, DEGRADED, DOWN
2. New model Source:
   - id: Int @id @default(autoincrement())
   - type: ArticleSource (OTS_AT | RSS only — MANUAL sources are not polled)
   - url: String @unique
   - enabled: Boolean @default(true)
   - pollIntervalMinutes: Int @default(15)
   - consecutiveFailures: Int @default(0)
   - lastSuccessAt: DateTime?
   - healthStatus: SourceHealth @default(OK)
   - createdAt: DateTime @default(now())
   - updatedAt: DateTime @updatedAt
   - runs: IngestionRun[]
   - @@index([enabled])
3. New model IngestionRun:
   - id: Int @id @default(autoincrement())
   - source: Source @relation(fields: [sourceId], references: [id])
   - sourceId: Int
   - startedAt: DateTime @default(now())
   - finishedAt: DateTime?
   - itemsFound: Int?
   - itemsNew: Int?
   - error: String?
   - @@index([sourceId, startedAt])
4. Add to Article model: contentHash String? @unique (SHA-256 of normalized title+body)

Then run: npx prisma migrate dev --name ingestion

This generates prisma/migrations/20260321_ingestion/migration.sql (the exact directory name from migrate dev may differ — use whatever Prisma generates). Commit the migration file. The pgLite setup-db.ts will automatically pick it up on the next test run.

Do NOT run prisma generate separately — migrate dev runs it automatically.
  </action>
