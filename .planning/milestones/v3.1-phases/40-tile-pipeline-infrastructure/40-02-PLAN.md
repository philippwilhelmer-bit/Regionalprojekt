    Implement in mapgen.ts:

    1. `stitchTiles(tiles: Buffer[][], cols: number, rows: number): Promise<Buffer>` — create a blank canvas (cols*256 x rows*256), composite all tile buffers at their grid positions using `sharp({ create: {...} }).composite([...])`, then `.resize(1200, 630, { fit: 'cover', position: 'centre' })`, then composite the attribution SVG via `buildAttributionSvg(1200, 630)`, then `.jpeg({ quality: 80 }).toBuffer()`. Note: the attribution composite must happen AFTER the resize/crop (so it's positioned relative to final 1200x630 output, not the raw tile canvas).

    Correction to stitching approach: sharp pipeline is sequential, so:
    ```
    Step 1: sharp({ create canvas }).composite(tiles) => toBuffer()
    Step 2: sharp(step1Buffer).resize(1200, 630, cover, centre) => toBuffer()
    Step 3: sharp(step2Buffer).composite([{ input: attributionSvg }]).jpeg({ quality: 80 }) => toBuffer()
    ```

    2. `uploadToBlob(articleId: number, imageBuffer: Buffer): Promise<string>` — call `put('maps/article-${articleId}.jpg', imageBuffer, { access: 'public', contentType: 'image/jpeg', allowOverwrite: true })`. Return `blob.url`.

       - Upload: `uploadToBlob(articleId, buffer)`.
       - Return `{ url, credit: '© basemap.at' }`.
       - Entire body wrapped in try/catch. On any error: `console.warn('[mapgen] article id=${articleId} -- ${error message} -- lat=${lat} lon=${lon}')` and return null.
