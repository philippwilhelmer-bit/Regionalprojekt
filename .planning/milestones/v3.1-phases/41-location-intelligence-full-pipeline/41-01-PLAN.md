  <action>
Create `src/lib/images/geocode.ts`:

1. Export `GeocodingResult` interface: `{ lat: number, lon: number, locationType: string, displayName: string }`.

2. Define `NominatimResult` interface (internal): `{ lat: string, lon: string, type: string, display_name: string }` (note: lat/lon are STRINGS in Nominatim jsonv2).

3. Import `GEOCODING_QUERY_OVERRIDE` from `./locextract`.

4. Implement `geocodeLocation(db: PrismaClient, placeName: string): Promise<GeocodingResult | null>`:
   - Apply override: `const query = GEOCODING_QUERY_OVERRIDE[placeName] ?? placeName`
   - Normalize cache key: `const normalized = query.trim().toLowerCase()`
   - Cache lookup: `db.geocodingCache.findUnique({ where: { normalizedName: normalized } })`
   - If cached, return mapped result immediately
   - On cache miss, fetch `https://nominatim.openstreetmap.org/search?q=${encodeURIComponent(query)}&countrycodes=at&format=jsonv2&limit=1` with header `User-Agent: Wurzelwelt/1.0 (https://wurzelwelt.at)`
   - If !res.ok, throw Error
   - Parse results, if empty return null
   - parseFloat(lat) and parseFloat(lon)
   - Use `db.geocodingCache.upsert()` to store (handles P2002 race condition):
     - where: { normalizedName: normalized }
     - create: { normalizedName, displayName, lat, lon, locationType }
     - update: {} (no-op if already exists)
   - Return GeocodingResult

Create `src/lib/images/geocode.test.ts`:

   - Test all behaviors listed above
   - For the "no HTTP call on second invocation" test: verify fetch mock was NOT called when findUnique returns a cached row
